# CR-14 開發日誌 — 草稿版本化管理

**日期**: 2026-05-07
**對應需求**: change-requirements-8.md CR-14

---

## 目標

將草稿備份系統由「每個檔案只保留最新一份」升級為「版本化保存」：

| 維度 | 舊行為 | 新行為 |
| ---- | ------ | ------ |
| 每檔案版本數 | 1（同一 fileId 直接覆寫） | 最多 40 版（新增時前插，超過截斷最舊） |
| 最多追蹤檔案數 | 20（超過淘汰最舊的那筆記錄） | 20（超過淘汰「最舊版本時間戳最早」的檔案） |
| DraftDialog UI | 兩欄（檔案列表 + 預覽） | 三欄（檔案列表 + 版本列表 + 預覽） |

---

## 實作細節

### 1. 資料結構重設計（`localDrafts.ts`）

#### 舊介面

```ts
interface LocalDraft {
  fileId: string;
  fileName: string | null;
  content: string;
  savedAt: number;
}
// localStorage: LocalDraft[]
```

#### 新介面

```ts
interface DraftVersion {
  content: string;
  savedAt: number;
}

interface FileDrafts {
  fileId: string;
  fileName: string | null;
  versions: DraftVersion[]; // 降冪排列（最新在前），最多 40 筆
}
// localStorage: FileDrafts[]
```

同一個 Storage key（`jpeditor.localDrafts`）沿用，不需要 key 遷移。

#### 常數

```ts
const MAX_FILES    = 20;  // 原 MAX_DRAFTS
const MAX_VERSIONS = 40;
```

---

### 2. `getFileDrafts()` — 含舊格式遷移

原 `getDrafts()` 重新命名為 `getFileDrafts()`。讀取時透過以下方式判斷是否為舊格式：

```ts
function isLegacyFormat(arr: unknown[]): arr is LegacyDraft[] {
  return arr.length > 0 && 'content' in (arr[0] as object);
}
```

舊格式（每筆有 `content` 欄位）自動以 `migrateLegacy()` 轉換後**立即寫回** localStorage，使遷移只發生一次：

```ts
function migrateLegacy(legacy: LegacyDraft[]): FileDrafts[] {
  const migrated = legacy.map((d) => ({
    fileId: d.fileId,
    fileName: d.fileName,
    versions: [{ content: d.content, savedAt: d.savedAt }],
  }));
  localStorage.setItem(LS_KEY, JSON.stringify(migrated));
  return migrated;
}
```

每個舊草稿會以「單一版本」的形式保留，既有備份不會遺失。

---

### 3. `saveDraft()` — 版本化插入邏輯

```
saveDraft(fileId, fileName, content)
  ├─ 讀取 allFiles
  ├─ 找到對應 fileId 的 FileDrafts？
  │   ├─ 是 → 前插新 DraftVersion；截斷至 MAX_VERSIONS
  │   └─ 否 → 建立新 FileDrafts，加入 allFiles
  │           若 allFiles.length > MAX_FILES
  │             → 找出「最舊版本（versions 尾端）的 savedAt」最小的那筆 FileDrafts
  │             → 以 splice 移除
  └─ 寫回 localStorage
```

淘汰策略選擇「最舊版本的時間戳最早的那個檔案」，語義為「整體上最久沒有備份活動的檔案」，與直覺上「最不活躍」的定義一致。

---

### 4. `removeVersion()` — 新增單版本刪除

```ts
export function removeVersion(fileId: string, savedAt: number): void {
  const allFiles = getFileDrafts();
  const idx = allFiles.findIndex((f) => f.fileId === fileId);
  if (idx < 0) return;
  allFiles[idx].versions = allFiles[idx].versions.filter(
    (v) => v.savedAt !== savedAt,
  );
  // 若版本清空，同時移除整個 FileDrafts 項目
  const updated =
    allFiles[idx].versions.length === 0
      ? allFiles.filter((_, i) => i !== idx)
      : allFiles;
  localStorage.setItem(LS_KEY, JSON.stringify(updated));
}
```

### 5. UI 三欄化（`DraftDialog.tsx`）

原為兩欄（左：`LocalDraft[]` 列表；右：預覽），改為三欄：

| 欄位 | 內容 | 寬度 |
| ---- | ---- | ---- |
| 左（`.list`） | `FileDrafts[]` 檔案列表，顯示檔名與版本數 | 200px |
| 中（`.versionList`） | 所選檔案的 `DraftVersion[]`，顯示備份時間 | 160px |
| 右（`.preview`） | 所選版本的內容預覽與「套用至編輯器」操作 | flex: 1 |

#### State 設計

```ts
const [fileDrafts, setFileDrafts]       = useState<FileDrafts[]>([]);
const [selectedFileId, setSelectedFileId]   = useState<string | null>(null);
const [selectedSavedAt, setSelectedSavedAt] = useState<number | null>(null);
```

從 `fileDrafts` 派生選取物件，不額外儲存冗餘資料：

```ts
const selectedFile    = fileDrafts.find((f) => f.fileId === selectedFileId) ?? null;
const selectedVersion = selectedFile?.versions.find((v) => v.savedAt === selectedSavedAt) ?? null;
```

#### 刪除後選取修正邏輯

- **刪除檔案**（`handleRemoveFile`）：若刪除的是當前選取的檔案，自動選取剩餘清單的第一筆及其最新版本。
- **刪除版本**（`handleRemoveVersion`）：若刪除的是當前選取的版本，嘗試選取同一檔案的第一個版本；若該檔案已無版本（`FileDrafts` 被一併移除），則跳至清單的第一個檔案。

#### 檔案列表排序

```ts
function sortedFileDrafts(list: FileDrafts[]): FileDrafts[] {
  return [...list].sort((a, b) => {
    const aTop = a.versions[0]?.savedAt ?? 0;
    const bTop = b.versions[0]?.savedAt ?? 0;
    return bTop - aTop;
  });
}
```

依「最新版本的時間戳」降冪，確保最近有備份的檔案顯示在上方。

## 設計決策

| 決策 | 說明 |
| ---- | ---- |
| 遷移在讀取時執行，立即寫回 | 確保遷移僅發生一次；舊頁籤若同時開啟也不會重複遷移 |
| 版本以「前插」而非「尾加」儲存 | 避免每次排序，直接以索引 0 取最新版本 |
| 淘汰「最舊版本時間戳最早」的檔案 | 語義最直覺：移除最久未有備份活動的檔案，而非最早建立的檔案 |
| 選取狀態以 `fileId` + `savedAt` 二元組表達 | 避免儲存整個物件參考；`savedAt` 在同一檔案內唯一（毫秒精度足夠） |
| `onApply`、`EditorLayout` 介面不變 | 最小化影響範圍，降低回歸風險 |
