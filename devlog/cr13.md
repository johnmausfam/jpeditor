# CR-13 開發日誌 — 加強 Loading 過場體驗

**日期**: 2026-05-05
**對應需求**: change-requirements-7.md CR-13

---

## 目標

在兩個位置新增 Loading 視覺回饋，提升使用者對非同步操作進行中狀態的感知：

| 位置 | 觸發時機 | 效果 |
| ---- | -------- | ---- |
| Header 下緣（全局進度條） | 儲存文件（`isSaving`）或讀取文件（`isOpeningFile`）| Indeterminate 橫向掃描條 |
| DrivePanel 文件列表 | Drive 文件列表讀取中（`isLoadingFiles`） | 骨架屏（Skeleton） |

---

## 實作細節

### 1. 新增 `isOpeningFile` state（`EditorLayout.tsx`）

原本的 `handleOpenDriveFile` 沒有對應的 loading state，僅靠 DrivePanel 內的 `openingFileId` 追蹤個別文件的開啟狀態，無法驅動全局進度條。新增本地 state：

```ts
const [isOpeningFile, setIsOpeningFile] = useState(false);
```

在 `handleOpenDriveFile` 加入 try/finally 結構，確保無論成功或失敗都會重設：

```ts
setIsOpeningFile(true);
try {
  const content = await downloadFileContent(accessToken, fileId);
  // ... 載入邏輯 ...
} catch (e) {
  // ... 錯誤處理 ...
} finally {
  setIsOpeningFile(false);
}
```

`handleOpenSharedFile` 透過呼叫 `handleOpenDriveFile`，自動繼承此行為，無需額外修改。

---

### 2. 全局進度條（`EditorLayout.tsx` + `EditorLayout.module.css`）

**JSX 放置位置**：`</header>` 的正下方（兄弟元素），作為獨立的 flex 列佔位，不干擾 Header 的 flex 布局：

```tsx
{/* ── CR-13: Global progress bar (below header) ── */}
{(isSaving || isOpeningFile) && (
  <div className={styles.progressBar} aria-hidden="true" />
)}
```

`aria-hidden="true"` 避免螢幕閱讀器播報裝飾性動畫元素。

**CSS 設計**：偽元素 `::after` 承擔動畫色條，父元素設 `overflow: hidden` 形成裁切框：

```css
.progressBar {
  height: 3px;
  background: #0f172a;   /* 深色底，融入 header 風格 */
  flex-shrink: 0;
  overflow: hidden;
  position: relative;
}

.progressBar::after {
  content: '';
  position: absolute;
  top: 0;
  left: -40%;
  width: 40%;
  height: 100%;
  background: #38bdf8;   /* sky-400，與深色主題形成高對比 */
  animation: progressSlide 1s ease-in-out infinite;
}

@keyframes progressSlide {
  0%   { left: -40%; }
  100% { left: 100%; }
}
```

選用偽元素而非獨立 `<span>` 子元素，減少 DOM 節點數量。動畫週期 1s（比骨架屏略快），傳達較高的操作緊迫感。

---

### 3. DrivePanel 骨架屏（`DrivePanel.tsx` + `DrivePanel.module.css`）

**JSX 改動**：將 `isLoadingFiles` 分支從純文字改為骨架屏，固定顯示 5 列：

```tsx
// 改動前
{isLoadingFiles && (
  <p className={styles.statusMsg} aria-live="polite">
    載入中…
  </p>
)}

// 改動後
{isLoadingFiles && (
  <div
    className={styles.skeleton}
    aria-busy="true"
    aria-label="載入中"
  >
    {Array.from({ length: 5 }).map((_, i) => (
      <div key={i} className={styles.skeletonItem} />
    ))}
  </div>
)}
```

`aria-busy="true"` 讓輔助技術得知容器內容正在更新；`aria-label="載入中"` 補充說明，取代原本的 `aria-live`。

**CSS 設計**：shimmer 效果採用背景漸層位移動畫，骨架列高度 57px 與實際 `.fileItem`（`padding: 10px` × 2 + 兩行文字高度）對齊，確保骨架與真實列表視覺高度一致：

```css
.skeleton {
  padding: 4px 0;
}

.skeletonItem {
  height: 57px;
  background: linear-gradient(
    90deg,
    #f1f5f9 25%,
    #e2e8f0 50%,
    #f1f5f9 75%
  );
  background-size: 200% 100%;
  border-bottom: 1px solid #f8fafc;
  animation: shimmer 1.2s ease-in-out infinite;
}

@keyframes shimmer {
  0%   { background-position: 200% center; }
  100% { background-position: -200% center; }
}
```

`background-size: 200%` 搭配位置從 `200%` 移動到 `-200%`，讓亮帶從右側掃向左側（即「光澤划過」的視覺效果）。週期 1.2s 比進度條略慢，模擬背景靜默等待的語意。

## 設計決策

| 決策 | 說明 |
| ---- | ---- |
| 不新增 Zustand 欄位 | `isOpeningFile` 屬於純 UI 狀態，不需跨元件共享，保留在 `EditorLayout` 本地 |
| 進度條不需淡出動畫 | `{condition && <div />}` 直接卸載元素，避免過多視覺噪音 |
| 骨架列數固定為 5 | 不依賴實際文件數，避免空資料夾時骨架與內容數量落差造成視覺跳動 |
| 純 CSS 動畫 | 不使用 JavaScript `requestAnimationFrame`，對渲染效能零負擔 |
