# CR-10 開發日誌 — 多工作目錄管理

**日期**: 2026-05-05
**對應需求**: change-requirements-5.md CR-10

---

## 目標

將原本只支援單一 `folderId` 的設定架構，擴充為可管理多組工作目錄的機制，並在 `DrivePanel` 提供下拉選單切換，最後一次使用的目錄以 localStorage 持久化。

---

## 資料模型設計

### `WorkFolder` 介面（`driveStore.ts`）

```ts
export interface WorkFolder {
  label: string;     // 使用者自訂的顯示名稱
  folderId: string;  // Google Drive 資料夾 ID
}
```

### `DriveState` 欄位異動

| 異動 | 舊 | 新 |
| ---- | -- | -- |
| 工作目錄資料 | `folderId: string` | `workFolders: WorkFolder[]` |
| 作用中目錄 | 同上 | `activeFolderId: string` |
| 設定方法 | `setFolderId(id)` | `setWorkFolders(folders)` / `setActiveFolderId(id)` |

## 實作細節

### 1. localStorage 鍵設計

```ts
const LS_FOLDER_LEGACY = 'jpeditor_drive_folder'; // 棄用，僅供遷移
const LS_FOLDERS       = 'jpeditor_drive_folders'; // WorkFolder[] JSON
const LS_ACTIVE        = 'jpeditor_drive_active';  // 最後使用的 folderId
```

### 2. 向後相容遷移（`initWorkFolders`）

store 初始化時呼叫 `initWorkFolders()`，以展開語法（`...initWorkFolders()`）注入初始狀態，確保遷移邏輯集中且只執行一次：

```ts
function initWorkFolders(): { workFolders: WorkFolder[]; activeFolderId: string } {
  const foldersRaw = localStorage.getItem(LS_FOLDERS);
  let workFolders: WorkFolder[] = [];

  if (foldersRaw) {
    try {
      workFolders = JSON.parse(foldersRaw) as WorkFolder[];
    } catch {
      workFolders = [];
    }
  } else {
    const legacy = localStorage.getItem(LS_FOLDER_LEGACY);
    if (legacy) {
      // 遷移舊資料：包裝成單筆 WorkFolder
      workFolders = [{ label: '預設目錄', folderId: legacy }];
      localStorage.setItem(LS_FOLDERS, JSON.stringify(workFolders));
    }
  }

  const savedActive = localStorage.getItem(LS_ACTIVE);
  // 確認 savedActive 仍在清單中，否則回退至第一筆
  const validActive =
    savedActive && workFolders.some((f) => f.folderId === savedActive)
      ? savedActive
      : (workFolders[0]?.folderId ?? '');

  return { workFolders, activeFolderId: validActive };
}
```

遷移邏輯分支：

```
localStorage 狀態           → 結果
────────────────────────────────────────────
jpeditor_drive_folders 存在  → 直接解析使用
jpeditor_drive_folder 存在   → 遷移為單筆 WorkFolder，寫入新鍵
兩者皆無                     → workFolders = []
```

`activeFolderId` 的回退策略：`savedActive` 若不在清單中（例如使用者刪除了該目錄），自動回退至 `workFolders[0]?.folderId`，避免指向不存在的目錄。

### 3. `setWorkFolders` / `setActiveFolderId`

```ts
setWorkFolders: (folders) => {
  localStorage.setItem(LS_FOLDERS, JSON.stringify(folders));
  set({ workFolders: folders });
},

setActiveFolderId: (id) => {
  localStorage.setItem(LS_ACTIVE, id);
  set({ activeFolderId: id });
},
```

### 4. `SettingsDialog` — 多目錄管理 UI

#### 本地狀態設計

Dialog 使用本地 `folderList: WorkFolder[]` state，對話框開啟時從 store 深拷貝（`workFolders.map((f) => ({ ...f }))`），避免直接修改 store 資料。關閉（取消）時本地變更自動丟棄，僅在點擊「儲存設定」時才寫回 store：

```ts
useEffect(() => {
  if (open) {
    setFolderList(
      workFolders.length > 0
        ? workFolders.map((f) => ({ ...f }))
        : [{ label: '', folderId: '' }],
    );
  }
}, [open, workFolders]);
```

#### 欄位更新用泛型 handler

```ts
const handleFolderChange = (
  index: number,
  field: keyof WorkFolder,
  value: string,
) => {
  setFolderList((prev) =>
    prev.map((f, i) => (i === index ? { ...f, [field]: value } : f)),
  );
};
```

`keyof WorkFolder` 確保 `field` 只能是 `'label'` 或 `'folderId'`，型別安全。

#### 儲存時的驗證與 `activeFolderId` 回退

```ts
const handleSave = () => {
  // 過濾掉任一欄位為空的列
  const valid = folderList.filter(
    (f) => f.label.trim() !== '' && f.folderId.trim() !== '',
  );
  const saved = valid.length > 0 ? valid : [{ label: '', folderId: '' }];
  setWorkFolders(saved);

  // 若目前作用中的目錄已被刪除，切換至第一筆
  const stillValid = saved.some((f) => f.folderId === activeFolderId);
  if (!stillValid) {
    setActiveFolderId(saved[0]?.folderId ?? '');
  }
  // ...
};
```

#### 刪除按鈕停用條件

清單僅剩一筆時停用刪除按鈕，保持至少一列可編輯（即使是空列）：

```tsx
<button
  disabled={folderList.length <= 1}
  onClick={() => handleRemoveFolder(index)}
>
  🗑
</button>
```

#### 移除 Enter 鍵儲存

原單一輸入框時支援 Enter 送出；多列表單中 Enter 鍵語意不明確（可能是換行至下一欄），故移除，僅保留 Escape 關閉。

### 5. `DrivePanel` — 工作目錄下拉選單

原本顯示原始 folder ID 字串的 `<p>` 段落，替換為 `<select>` 下拉選單：

```tsx
<select
  className={styles.folderSelect}
  value={activeFolderId}
  onChange={(e) => handleFolderChange(e.target.value)}
  aria-label="切換工作目錄"
>
  {workFolders.map((f) => (
    <option key={f.folderId} value={f.folderId}>
      📂 {f.label}
    </option>
  ))}
</select>
```

切換時的 `handleFolderChange` 同時清空搜尋關鍵字、觸發 `onRefresh`，確保列表即時反映新目錄內容：

```ts
const handleFolderChange = (folderId: string) => {
  setActiveFolderId(folderId);
  setKeyword('');
  onRefresh();
};
```

未設定任何工作目錄時（`workFolders.length === 0`），顯示提示文字而非空的下拉選單。
