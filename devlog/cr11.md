# CR-11 開發日誌 — 文件分享功能（Share URL）

**日期**: 2026-05-05
**對應需求**: change-requirements-6.md CR-11

---

## 目標

新增「🔗 分享」按鈕，讓使用者一鍵產生帶有 `fileId` 的分享網址並複製至剪貼簿；收到連結的人開啟後，編輯器會自動觸發 OAuth 登入並載入指定文件，載入完成後清除 URL 中的 queryString。

---

## 實作細節

### 1. 新增 state 與 ref（`EditorLayout.tsx`）

```ts
const [shareCopied, setShareCopied] = useState(false);
const shareLinkRef = useRef<{ fileId: string; fileName: string } | null>(null);
```

- `shareCopied`：控制按鈕短暫顯示「✓ 已複製！」的 2 秒視覺回饋
- `shareLinkRef`：在使用者尚未登入時，暫存分享連結的 `fileId` / `fileName`，避免在非同步登入流程結束前遺失資訊（不用 state 是為了不觸發重新渲染）

---

### 2. `handleOpenSharedFile`（包裝 `handleOpenDriveFile`）

```ts
const handleOpenSharedFile = useCallback(
  async (fileId: string, fileName: string) => {
    await handleOpenDriveFile(fileId, fileName);
    history.replaceState(null, '', window.location.pathname);
  },
  [handleOpenDriveFile],
);
```

與現有的 `handleOpenDriveFile` 行為完全一致（下載、載入編輯器、進入 preview 模式），差異在於載入成功後呼叫 `history.replaceState` 清除 queryString，防止使用者重新整理頁面時重複載入。

#### handler 宣告順序

`handleLogin` 的 deps 陣列包含 `handleOpenSharedFile`，因此必須確保宣告順序為：

```
handleOpenDriveFile → handleOpenSharedFile → handleLogin
```

原本順序相反（`handleLogin` → `handleLogout` → `handleOpenDriveFile`），調整後消除了 TypeScript「used before its declaration」TDZ 錯誤。

---

### 3. `handleLogin` 分岔處理

登入成功後依 `shareLinkRef.current` 判斷是否為分享連結開啟：

```ts
if (shareLinkRef.current) {
  const { fileId, fileName } = shareLinkRef.current;
  shareLinkRef.current = null;
  await handleOpenSharedFile(fileId, fileName);
} else {
  setDrivePanelOpen(true);
  const { activeFolderId } = useDriveStore.getState();
  if (activeFolderId) await handleLoadDriveFiles();
}
```

- 分享連結開啟：直接載入指定文件，不顯示 DrivePanel
- 一般登入：維持原本行為（開啟 DrivePanel 並載入文件列表）

---

### 4. mount 時的 queryString 偵測（useEffect）

```ts
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const fileId = params.get('fileId');
  if (!fileId) return;
  const fileName = params.get('fileName') ?? '';
  shareLinkRef.current = { fileId, fileName };
  if (useDriveStore.getState().isLoggedIn) {
    shareLinkRef.current = null;
    handleOpenSharedFile(fileId, fileName);
  } else {
    handleLogin();
  }
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, []);
```

deps 陣列為 `[]`（僅執行一次），避免 `handleLogin` / `handleOpenSharedFile` 的 identity 變更造成重複觸發。ESLint exhaustive-deps 警告以 disable comment 抑制，因為這是刻意的 mount-only effect。

---

### 5. `handleShare`（複製分享網址）

```ts
const handleShare = useCallback(async () => {
  if (!currentFileId) return;
  const params = new URLSearchParams({ fileId: currentFileId });
  if (currentFileName) params.set('fileName', currentFileName);
  const url = `${window.location.origin}${window.location.pathname}?${params.toString()}`;
  try {
    await navigator.clipboard.writeText(url);
    setShareCopied(true);
    setTimeout(() => setShareCopied(false), 2000);
  } catch {
    window.prompt('複製此連結以分享：', url);
  }
}, [currentFileId, currentFileName]);
```

- `URLSearchParams` 自動對 `fileName` 進行 percent-encode，無需手動呼叫 `encodeURIComponent`
- `navigator.clipboard` 在非 HTTPS 環境（localhost 除外）可能被瀏覽器拒絕，catch 後降級至 `window.prompt`，讓使用者手動複製

---

### 6. Header 分享按鈕（JSX）

```tsx
{isLoggedIn && (
  <button
    className={styles.headerBtn}
    onClick={handleShare}
    disabled={!currentFileId}
    title={currentFileId ? '複製分享連結' : '請先儲存文件'}
    aria-label="分享"
  >
    {shareCopied ? '✓ 已複製！' : '🔗 分享'}
  </button>
)}
```

- 位置：Header 右側，「💾 儲存」按鈕左邊
- `disabled={!currentFileId}`：文件未存檔至 Drive 時禁用
- `title` 提示隨狀態切換，提供說明

## 同步實作：CR-12 — `assertOk` 強化（`googleDriveApi.ts`）

分享連結開啟時若遭遇 403（文件未共用），錯誤訊息需對使用者友善，因此 CR-12 一併於本次實作：

```ts
async function assertOk(res: Response): Promise<void> {
  if (!res.ok) {
    if (res.status === 401) {
      throw new Error('認證已過期，請重新登入。');
    }
    if (res.status === 403) {
      throw new Error(
        '您沒有存取此文件（或資料夾）的權限。請確認該項目已與您的 Google 帳號共用。',
      );
    }
    if (res.status === 404) {
      throw new Error('找不到指定的文件，它可能已被刪除或移至其他位置。');
    }
    const body = await res.text().catch(() => '');
    throw new Error(`Drive API ${res.status}: ${body.slice(0, 300)}`);
  }
}
```

修改範圍僅限 `assertOk` 函式，所有呼叫端（`listMarkdownFiles`、`downloadFileContent`、`createDriveFile`、`updateDriveFile`）自動受益。
